How this dashboard fits together

RepNet is the only write path going forward. Excel files keep their shape and continue driving your pivots. Five external systems plug in as event sources. Six Azure Functions do the heavy lifting in the background.

1 Read both Excel files via Graph

RepNet reads REPO-Q006 Ticketing Log V2 + PARTS TRACKER at view-open via Microsoft Graph Excel REST. 5-min in-browser cache, manual ↻ refresh. Existing pivots, dashboards, formulas keep working untouched.

2 Write back on form submit

"+ New Ticket" / "+ Parts Dispatch" / "Mark for return" append rows or PATCH cells via Excel REST API. Zero schema change. Manual edits to Excel still possible as emergency override.

3 FedEx parts auto-tracking

Azure Function parts-fedex-poll · every 30 min. Reads Part Tracker rows where Delivered is blank → calls FedEx REST → writes back delivery status, signature, ETA.

4 Maxoptra ↔ RepNet bridge

Azure Function maxoptra-sync · every 15 min. Polls Maxoptra Order API for jobs tagged with a RepNet Ticket No → updates ticket Factory Status (Collected → Returned → In Production → Out for delivery → Delivered).

5 iAuditor inspection capture

Phase 1: engineer exports inspection PDF → uploads on ticket. Phase 2: Azure Function iauditor-pull auto-fetches by template + REP No reference and attaches to the ticket.

6 Auto-emails via Graph

"Mark for return" → email to transport@reposefurniture.co.uk. Customer notification email when collection booked. Same Graph /me/sendMail pattern used for Document Control.

7 Weekly digest email · Mondays 06:00

Azure Function service-weekly-digest sends an HTML summary every Monday: last week's opens/closes, top fault, overdue list, repeat returns, parts spend. Recipients: service team + senior managers.

8 CAPA auto-bridge

When the same fault code repeats N times in 30 days for the same model, RepNet flags a CAPA suggestion. One click opens a pre-filled CAPA form in the existing CAPA module — closes the loop ISO-style.

9 Quality module cross-link

When mechanism-fault count breaches a Quality SPC limit (e.g. >5 in a week), the Service tab surfaces a link straight to the relevant Quality dashboard chart. Bidirectional — a Quality-flagged trend can drill back to its source tickets.

10 Customer-facing tracking page

Public read-only URL repose.tracking/t/{token}. No login. Customers see a friendly 5-stage timeline matching their ticket's progress. Reduces "where's my chair?" calls. Token-secured; no internal data exposed.

11 Forecast & SLA pre-alerts

13-month trend predicts next-month opens + parts spend. SLA pre-alerts fire when a ticket is 80%+ of its proposed-close window with no recent action — surfaced in the alert banner so issues get caught before they breach.

12 Service engineer schedule

Pulls Maxoptra collection/delivery jobs + iAuditor scheduled audits + manual on-site visits into one calendar. Engineer sees the whole week at a glance; clicking an event opens the linked ticket.
